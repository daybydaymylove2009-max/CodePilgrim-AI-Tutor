## 项目愿景

CodePilgrim 是一款专为**非理工背景学习者**设计的沉浸式编程伴学系统。我们相信：

> 编程不是"看教程→背语法→做习题"的教条化过程，而是在真实场景中探索、试错、创造的沉浸式体验。

### 核心问题

市面上的编程课程普遍存在"看着会、写就懵"的痛点：

```
❌ 假学习：跟着教程敲代码 → 关闭教程 → 什么都写不出来
✅ 真掌握：关闭教程 → 能独立写出程序 → 遇到新问题知道从何入手
```

CodePilgrim 的目标不是让你"完成课程"，而是让你**真正掌握编程能力**。

## 核心设计理念

### Vibe 为帆，代码为舵

> **Vibe 是风，助你启航；代码是舵，由你掌控。**
>
> 借 Vibe 之力入门，以真实力驾驭程序——你才是代码的主人。

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│   Vibe 是辅助，不是替代                                       │
│                                                              │
│   · Vibe 帮你推开门    → 但走进去的是你自己                    │
│   · Vibe 帮你跑第一段  → 但跑完全程的是你自己                  │
│   · Vibe 帮你看见可能  → 但把可能变成现实的是你自己            │
│                                                              │
│   ─────────────────────────────────────────────────────────   │
│                                                              │
│   真正的掌握：                                                │
│                                                              │
│   不依赖 AI，你也能写出代码                                    │
│   不依赖教程，你也能解决问题                                   │
│   不依赖提示，你也能独立创造                                   │
│                                                              │
│   程序的编写、运行、调试——每一步都出自你手                      │
│   这才是"做程序的主人"                                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**从 Vibe 到 Mastery 的旅程**：

| 阶段 | 你在做什么 | AI 在做什么 | 你的状态 |
|------|-----------|-----------|---------|
| **Vibe** 体验 | 用自然语言描述想法 | 帮你生成代码 | "哇，我做出来了！" |
| **Understand** 理解 | 读懂每一行代码 | 逐行讲解 | "原来它是这样做的" |
| **Modify** 改造 | 独立修改和扩展代码 | 仅在需要时建议 | "我可以改它！" |
| **Mastery** 掌握 | 独立编写完整程序 | 关闭，等你召唤 | "我自己也能写！" |

### 沉浸式学习，拒绝教条化

| 维度 | 传统教条化 | CodePilgrim 沉浸式 |
|------|-----------|-------------------|
| 学习方式 | 看教程→背语法→做习题 | 在真实场景中"做中学" |
| 反馈时机 | 章节结束后统一测试 | 代码写完即时反馈 |
| 错误处理 | "错误！正确答案是..." | "这个思路很有趣...让我们看看还能怎么改进" |
| 进度衡量 | 完成了多少章节 | 解决了多少真实问题 |
| 独立能力 | 学完课程→仍是小白 | 学完就能独立项目 |

### "真掌握"三层验证机制

只有通过三层验证，才算真正"掌握"了一个知识点：

```
第一层：照着做 (Guided)    → AI 导师引导，逐步提示完成代码
第二层：独立做 (Independent) → AI 隐藏提示，无提示独立完成 > 60%
第三层：创新做 (Creative)   → 只有功能需求，无步骤提示，完成 > 50% 功能点
```

## 核心设计原则

### AI自主伴学为主，人类辅助为辅

```
┌──────────────────────────────────────────────────┐
│  AI 自主伴学（核心）                                │
│  · AI 导师引导学习全过程                            │
│  · AI 智能答疑 + 深度追问                          │
│  · AI 自动生成课件和试题                            │
│  · AI 评分 + 个性化反馈                            │
├──────────────────────────────────────────────────┤
│  人类教师（可选辅助，非必要不出现）                    │
│  · 视频参考：隐藏在二级菜单                          │
│  · 人工辅导：隐藏在"需要更多帮助"三级菜单             │
│  · 教师仪表盘：只读参考，非核心功能                   │
└──────────────────────────────────────────────────┘
```

### 游戏化设计原则

基于对Duolingo游戏化机制的深度分析，CodePilgrim确立了四条游戏化设计原则：

| 原则 | 描述 |
|------|------|
| **能力锚定** | 所有游戏化奖励必须与真实编程能力挂钩 |
| **渐进撤除** | 游戏化支持随能力增长而递减，从外在激励过渡到内在成就感 |
| **深度优先** | 不设时间压力，鼓励探索试错，奖励理解而非速度 |
| **内在转化** | 从"获得积分"到"解决问题"的成就感转化 |
